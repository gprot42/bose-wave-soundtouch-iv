## 4. Overview

The SoundTouch Web API provides HTTP-based control of Bose SoundTouch devices via port **8090** using **GET** and **POST**.

---

## 4.1 Special Types

### ART_STATUS
- INVALID
- SHOW_DEFAULT_IMAGE
- DOWNLOADING
- IMAGE_PRESENT

Other primitive types:
- BOOL: `"true"` / `"false"`
- INT: 32-bit integer
- UINT / UINT64
- STRING (XML-escaped)
- URL
- IPADDR
- MACADDR

---

## 5. General Status and Errors

Default response:
```xml
<status>STRING</status>
```

Error response:
```xml
<errors deviceID="STRING">
  <error value="INT" name="STRING" severity="STRING">STRING</error>
</errors>
```

## 6. API Endpoints

### 6.1 /key
Send remote key presses.

**POST**
```xml
<key state="press|release" sender="STRING">KEY_VALUE</key>
```

---

### 6.2 /select
Select an input source.

**POST**
```xml
<ContentItem source="AUX|BLUETOOTH|PRODUCT" sourceAccount="STRING"/>
```

---

### 6.3 /sources
List available sources.

**GET**
```xml
<sources>
  <sourceItem source="STRING" status="READY|UNAVAILABLE"/>
</sources>
```

---

### 6.4 /bassCapabilities

**GET**
```xml
<bassCapabilities>
  <bassAvailable>true</bassAvailable>
  <bassMin>-10</bassMin>
  <bassMax>10</bassMax>
</bassCapabilities>
```

---

### 6.5 /bass

**GET / POST**
```xml
<bass>INT</bass>
```

---

### 6.6 /getZone
Get multi-room zone state.

### 6.7 /setZone
Create multi-room zones.

### 6.8 /addZoneSlave
### 6.9 /removeZoneSlave

---

### 6.10 /nowPlaying
Get current playback state.

---

### 6.11 /trackInfo
Track metadata.

---

### 6.12 /volume

**GET**
```xml
<volume>
  <targetvolume>INT</targetvolume>
  <muteenabled>false</muteenabled>
</volume>
```

---

### 6.13 /presets
List presets.

---

### 6.14 /info
Device info.

---

### 6.15 /name
Set device name.

---

### 6.16 /capabilities
List supported capabilities.

---

### 6.17 /audiodspcontrols
Audio DSP modes.

---

### 6.18 /audioproducttonecontrols
Bass & treble.

---

### 6.19 /audioproductlevelcontrols
Speaker level control.

## 7. WebSockets

WebSocket notifications via port **8080**, protocol **gabbo**.

Example:
```js
new WebSocket("ws://DEVICE_IP", "gabbo")
```

Notifications include:
- NowPlayingChange
- VolumeChange
- BassChange
- ZoneMapChange
- PresetsChanged
- SourcesChanged
